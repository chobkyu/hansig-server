users
ㄴ 회원가입 시
    /post user 
        ㄴ 성공 시 201로 응답
        ㄴ 입력이 잘못 되었을 경우 400으로 응답
        ㄴ 중복값이 발생했을 경우 409로 응답한다
        


ㄴ 로그인
    /post login
        ㄴ 성공 시 201로 응답, 토큰 반환
        ㄴ 입력 값이 잘못 되었을 경우 400으로 응답

       

ㄴ 유저 정보 수정 시
    /post user/info
        ㄴ 성공 시 201로 응답
        ㄴ 입력 값이 잘못 되었을 경우 400으로 응답
        ㄴ 로그인이 안되어 있을 시 401로 응답
        ㄴ 사용자를 찾을 수 없을 시 404 //이건 좀 나중에
    ㄴ 비밀번호 변경 시
        ㄴ 성공 시 201 리턴
        
ㄴ 유저 정보 읽을 시
    /get user/info
        ㄴ 성공 시 200 리턴
        ㄴ 유저 없을 시 404 리턴
        ㄴ 잘못된 입력일 시 400
        ㄴ 중복 값이 발생했을 경우 409
    
owner
ㄴ 사업자로 회원가입 시
    /post owner
        ㄴ 성공 시 201로 응답
        ㄴ 입력이 잘못 되었을 경우 400으로 응답
        ㄴ 중복값이 발생했을 경우 409로 응답한다

ㄴ 사업자로 로그인 시
    /post owner/login
        ㄴ 성공 시 201로 응답, 토큰 반환
        ㄴ 입력 값이 잘못 되었을 경우 400으로 응답

hansigdang
ㄴ 한식 뷔페 리스트 조회
    /get hansic/all
        ㄴ 성공 시 한식 뷔페 리스트 조회(dto 추후 작성 예정) 상태코드 200 리턴
        ㄴ 성공 시 리스트 데이터 리턴
        ㄴ 성공 시 리스트 데이터에 dto 컬럼 확인

        ㄴ 리스트가 null 값이면 204 리턴
        ㄴ 실패 시 500 리턴

ㄴ 한식 뷔페 지역별 조회
    /get hansic/loc/:id
        ㄴ 성공 시 지역 별 한식 뷔페 리스트 조회 상태코드 200 리턴
        ㄴ 성공 시 리스트 데이터 리턴
        ㄴ 성공 시 리스트 데이터에 dto 컬럼 확인
        ㄴ 리스트 요소의 location_id는 요청 id 값과 같아야 한다
        
        
        ㄴ 실패 시
            ㄴ 잘못된 지역 id 입력 시 401 조회(number이 아닐 때)
                /*
                    지역 데이터는 다음과 같습니다. 데이터베이스는 클라우드 디비를 아직 구축하지 않아서 추후 공유드리겠습니다
                    {"서울", "인천", "김포", "대구", "세종", "부산", "경주", "광주", "대전", "성남", "전주", "울산"}
                    id는 서울부터 1입니다.
                */ 
            ㄴ 없는 지역 id 요청 시 404 조회
            ㄴ 실패 시 500 리턴

ㄴ 한식 뷔페 조회
    /get hansic/:id
        ㄴ 성공 시 해당 한식 뷔페 데이터 리턴
        ㄴ 성공 시 뷔페 데이터 dto 컬럼 확인

        ㄴ 실패 시
            ㄴ 해당 데이터를 찾을 수 없을 때는 404 리턴
            ㄴ 잘못된 id 입력 시 400 리턴
            ㄴ 실패 시 500 리턴
