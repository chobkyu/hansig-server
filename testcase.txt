ㄴ 회원가입 시
    /post user 
        ㄴ 성공 시 201로 응답
        ㄴ 입력이 잘못 되었을 경우 400으로 응답
        ㄴ 중복값이 발생했을 경우 409로 응답한다
        


ㄴ 로그인
    /post login
        ㄴ 성공 시 201로 응답, 토큰 반환
        ㄴ 입력 값이 잘못 되었을 경우 400으로 응답

       
ㄴ 지역 조회
    /get user/location 
        ㄴ 성공 시 지역 리스트로 응답
        ㄴ 성공 시 각 요소에 id와 location이 존재해야 하며 각 타입은 number와 string이여야 한다.
        ㄴ 배열의 길이는 12여야 한다. (db 조정 시 변경 예정)
        ㄴ 실패 시 500리턴

ㄴ 유저 정보 수정 시
    /patch user/info
        ㄴ 성공 시 201로 응답
        ㄴ 입력 값이 잘못 되었을 경우 400으로 응답
        ㄴ 로그인이 안되어 있을 시 401로 응답
        ㄴ 사용자를 찾을 수 없을 시 404 //이건 좀 나중에
    ㄴ 비밀번호 변경 시
        ㄴ 성공 시 201 리턴
        
ㄴ 유저 정보 읽을 시
    /get user/info
        ㄴ 성공 시 200 리턴
        ㄴ 유저 없을 시 404 리턴
        ㄴ 잘못된 입력일 시 400
        ㄴ 중복 값이 발생했을 경우 409
    
owner
ㄴ 사업자로 회원가입 시
    /post owner
        ㄴ 성공 시 201로 응답
        ㄴ 입력이 잘못 되었을 경우 400으로 응답
        ㄴ 중복값이 발생했을 경우 409로 응답한다

ㄴ 사업자로 로그인 시
    /post owner/login
        ㄴ 성공 시 201로 응답, 토큰 반환
        ㄴ 사업자가 아닐 시 401로 응답
        ㄴ 입력 값이 잘못 되었을 경우 400으로 응답
        ㄴ 입력 값이 다른 값이 있을 겅우 400으로 응답

ㄴ 메뉴 입력 시
    /post owner/menu
        ㄴ 성공 시 201로 응답
        ㄴ 실패 시
            ㄴ 입력 값이 잘못 되었을 경우 400으로 응답
            ㄴ 입력 값이 누락되었을 경우 400으로 응답
            ㄴ 다른 입력 값이 들어왔을 경우 400으로 응답

ㄴ 메뉴 조회 시
    /get owner/menu/:id
        ㄴ 성공 시 해당 식당 메뉴 리턴
        ㄴ 성공 시 배열 형태 리턴
        ㄴ 성공 시 메뉴 dto 컬럼 확인
        ㄴ 해당 식당 id는 요청 id와 일치해야 한다.
        ㄴ 리스트가 null이면 204 리턴
        ㄴ 실패 시
            없는 id면 404리턴
            
            

ㄴ 메뉴 삭제 시
    /delete owner/menu
        ㄴ 성공 시 204를 응답한다.
        ㄴ 실패 시 
            id가 숫자가 아닐 경우 400으로 응답한다.
            없는 메뉴 일 시 404 리턴
            권한이 없을 경우 401 리턴
            로그인 안되어 있을 시 401리턴

ㄴ 메뉴 수정 시
    /patch owner/menu
        ㄴ 성공 시 변경된 메뉴를 응답한다.
        ㄴ 입력이 누락 됐을 시 400을 응답한다.
        ㄴ 입력에 잘못된 값이 들어갔을 시 400을 응답한다.
        ㄴ 없는 메뉴일 시 404를 응답한다.
        ㄴ 이름이 중복일 경우 409로 응답한다.

ㄴ 리뷰 수정 시
    /patch review/update/:id
        ㄴ 성공 시 변경된 리뷰를 응답한다.
            review는 string, star 는 number, imgUrl는 Array<string>,
            user.userNickName은 string, user.id는 number여야 한다.
        ㄴ 입력이 누락 됐을 시 400을 응답한다.
        ㄴ 입력에 잘못된 값이 들어갔을 시 400을 응답한다.
        ㄴ 없는 리뷰일 시 404를 응답한다.
        ㄴ 리뷰 입력자가 아닐 시 401을 응답한다.


ㄴ 해당 리뷰 조회 시

            review는 string, star 는 number, imgUrl는 Array<string>,
            user.userNickName은 string, user.id는 number,
            reviewComment는 Array여야 한다.

        ㄴ 성공 시 해당 리뷰 댓글 리턴
        ㄴ 실패 시
            ㄴ 없는 id일 시 404로 응답
            ㄴ 잘못된 파라미터일 시 400으로 응답


ㄴ 리뷰 리스트 조회 시

        ㄴ 잘못된 파라미터일 시 400으로 응답

ㄴ 유저 별 리뷰 리스트 조회 시
